RAS

Assign roles to members based on reactions added to or removed from a message.

Before You Start:

  • Move Yui's role to the top of your server's role hierarchy.
  • Make sure one of Yui's roles has the Administrator permission.
  • Make sure Yui has access to all emotes you plan to add to your ras message. Yui has access to all default emotes, and any emotes from servers she is a member of.
  • Make sure all the roles you plan to add to your ras message are already created. Yui cannot create roles.

Usage:

y!ras create
You will be asked if you want to add ras to an existing message, or create a new one.
